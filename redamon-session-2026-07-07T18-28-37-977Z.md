# Penetration Test Summary Report

## 1. Summary
A black-box security assessment was conducted against the target web application at `http://lab-834136ae/` with the sole objective of recovering a hidden flag. A reflected cross-site scripting (XSS) vulnerability was identified in the `name` parameter of `/page`, which allowed the execution of arbitrary JavaScript in the browser. By crafting a payload that bypassed the server-side `<` character filter and preserved the exact `alert('XSS')` call required by the application’s challenge logic, the flag was successfully retrieved:  
`FLAG{64699b06af18be8aa11d1cffbaa67820d88f3b7d3b5d53ab95f9eff3796117d9}`

The assessment was completed in 9 iterations, culminating in a working exploitation phase. No credentials were discovered, and no persistent sessions were established beyond the stateless HTTP interaction.

## 2. Key Findings
- **Single-page application** running on `nginx/1.31.1`, titled “Dalgona - Triangle”, with minimal exposed endpoints (`/` and `/page`).
- The `/page` endpoint contains a form with a `name` parameter that is reflected directly into the HTML `<input>` element’s `value` attribute.
- The application implements a weak input filter that strips the `<` character but leaves many other dangerous characters intact (`"`, `'`, `` ` ``, `(`, `)`, `>`, etc.).
- No `Content-Security-Policy` or `X-XSS-Protection` headers were present in the response.
- The server-side logic evaluates whether the client performed an `alert('XSS')` call; any deviation (such as an empty alert or missing quotes) results in a failure message.
- The vulnerability was exploited using the payload:  
  `" autofocus onfocus=alert(`XSS`) "`  
  which broke out of the `value` attribute, added `autofocus` and `onfocus` event handlers, and used backtick template literals to preserve the exact `'XSS'` argument without conflicting with the surrounding HTML quote context.

## 3. Discovered Credentials
None. No brute-force or authentication testing was performed as the application did not require login credentials.

## 4. Sessions Established
No authenticated or persistent sessions were established. All interactions were stateless HTTP GET requests.

## 5. Vulnerabilities Found
| # | Vulnerability | Severity | Details |
|---|---------------|----------|---------|
| 1 | Reflected Cross-Site Scripting (XSS) in `name` parameter | High | The `name` parameter on `/page` is reflected into an HTML attribute without proper encoding. A filter removes `<`, but other special characters are allowed. An attacker can inject event handlers (e.g., `onfocus`) to execute arbitrary JavaScript in the context of the victim’s browser. This can lead to session hijacking, phishing, or defacement. The server additionally evaluates the alert content, making it possible to bypass challenge-based filters with backtick syntax. |

### Vulnerability Details
- **Endpoint:** `http://lab-834136ae/page?name=...`
- **Reflection context:** HTML attribute (`value="..."`)
- **Filter bypass:** `<` stripped, backtick (`` ` ``) and `"`/`'` unfiltered – allows attribute breakout via `autofocus onfocus=alert(`XSS`)`.
- **Proof-of-concept payload (URL-encoded):**  
  `%22%20autofocus%20onfocus%3Dalert(%60XSS%60)%20%22`
- **Impact:** Full client-side code execution; the flag was disclosed as a direct result of the exploitation.

## 6. Recommendations
- **Output encoding:** Apply contextual output encoding (e.g., HTML entity encoding for attribute values) to all user-controlled data reflected in the page.
- **Input validation:** Implement a strict allowlist of acceptable characters for the `name` parameter.
- **Content Security Policy:** Deploy a strong CSP header that restricts inline scripts and event handlers (`script-src 'none'`; consider `require-trusted-types-for 'script'`).
- **Remove debug/challenge logic:** The presence of server-side JavaScript evaluation based on user-supplied input (`alert('XSS')`) should be removed in production.
- **Security headers:** Add `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, and `Referrer-Policy` as part of defense-in-depth.
